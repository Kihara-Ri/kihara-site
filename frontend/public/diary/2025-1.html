大年初一，久违的算法焦虑
Leetcode 219.存在重复元素II
给你一个整数数组`nums`和一个整数`k`，判断数组中是否存在两个不同的索引`i`和`j`，满足`nums[i] == nums[j]`且`abs(i - j) <= k`。如果存在，返回`true`；否则，返回`false`。
这并不是一个难题，反而是一个不能再基础的题，但是只要一涉及到哈希表，我对其操作就懵了，现代编程语言的语法相当灵活，但我在写算法的时候仍然还用着最原始的操作
让我们来看一下Python下可以怎样操作：
```py
def containsNearbyDuplicate(self, nums: List[int], k: int) -> bool:
d = {}
for i, x in enumerate(nums):
if x in d and i - d[x] <= k:
return True
d[x] = i
return False
```
使用`enumerate()`可以解构出一个元组数组，带序号和数组对应位置的值，因此在这个循环中需要用到`i, x`来接收这个函数的返回值。python中的哈希表相当简单，直接使用一个字典即可，判断某个值是否在字典里可以直接使用`in`，而我们要在哈希表里创建一个键值对也很简单：`d[x] = i`。这类使用哈希表来降低复杂度的题我没少做，但总记不住各类编程语言中应该如何处理
在TypeScript中定义哈希表：`const d: Map<number, number> = new Map();`
查询哈希表中是否有某个元素：`d.has(nums[i])`，提取哈希表中的某个元素的值：`d.get(nums[i])`，如果没有这个值或提取失败则返回`null`，向哈希表中添加一个键值对：`d.set(nums[i], i)`
GO的语法和别家又不太一样，像是python和C++的杂交种，创建一个哈希表：`d := map[int]int{}`，看着有点晕，`[int]`表示的是key，后面那个没有括号的`int`表示value的类型，最后的`{}`表示的是初始化一个空的map，然后遍历切片`nums`时GO竟然又和python不一样，直接`for i, x := range nums {...}`就可以获得索引和值，在哈希表中创建一个键值对时又变得和python一样简单了
