格雷码的定义方式为每个邻近数字都只相差一个位元, 可以使装置做数字步进时只需要动最少的位元数, 以此提高稳定性

```text
0  　　 000    000
1  　　 001    001
2   　　011    010
3   　　010    011
4   　　110    100
5   　　111    101
6   　　101    110
7   　　100    111
```

对于格雷码G(n), 它一定是0, 1, 2, ..., \power(2, n) - 1 在二进制中的某个排列, 因为我们需要让这些数字全部都在编码中能够有一一对应, 只不过现在相比与一般的由小到大的排列, 顺序发生了变化, 实际上它们确实遵循着某种排列顺序, 只不过规律并不显而易见

又因为这个排列中相邻的元素只能相差一个位元, 因此开头和结尾也是相差一个的, 或者说你可以说这个编码会形成一个环, 并没有开头或结尾, 你可以从任何一个地方开始

G(n) = B(n+1) XOR B(n)

二进制数转格雷码

```c++
int BinaryToGray(int n) {
  return (n >> 1) ^ n;
}
```

格雷码转二进制数

```c++
int GrayToBinary(int g) {
  int mask = g;
  while (mask) {
    mask = mask >> 1;
    g = g ^ mask;
  }
  return g;
}
```

```text
初始: g = 0110, mask = 0110

① mask >>= 1 → 0011
   g ^= mask → 0110 ^ 0011 = 0101

② mask >>= 1 → 0001
   g ^= mask → 0101 ^ 0001 = 0100

③ mask >>= 1 → 0000 → 结束

返回 g = 0100 ✅
```

```text
格雷码:   g3 g2 g1 g0
          ↓  ↓  ↓  ↓
          0  1  1  0
           ↘ ↘ ↘
第一次传播:   0  1⊕1  1⊕0
                ↓     ↓
第二次传播:      0⊕1⊕1  1⊕1⊕0
结果(二进制):   0  1  0  0
```
